## 💡 Pro Tips

### 1. 个性化每次发送

**不要：** 群发相同邮件
**要做：** 每封邮件都针对收件人定制

**例如：**
```
给 Ondo：强调 OUSG/USDY 的具体用例
给 Paradigm：强调市场机会和回报
给技术团队：强调技术深度
给商务团队：强调商业价值
```

### 2. 渐进式披露

**不要：** 第一封邮件就发送所有文档
**要做：** 根据兴趣程度逐步提供

**顺序：**
```
首次接触：ONE_PAGER（激发兴趣）
表示兴趣：EXECUTIVE_BRIEF（详细了解）
技术尽调：TEST_REPORT + GitHub（证明能力）
投资尽调：COMPETITIVE_ANALYSIS（理解市场）
```

### 3. 保持更新

**创建版本控制：**
```
ILAL_EXECUTIVE_BRIEF_v1.0.md (2026-02-13)
ILAL_EXECUTIVE_BRIEF_v1.1.md (2026-03-01) - 添加新客户
ILAL_EXECUTIVE_BRIEF_v2.0.md (2026-06-01) - 主网上线
```

### 4. 准备不同格式

**Markdown（当前）：**
- 适合：GitHub, 技术团队
- 优点：易于更新，版本控制

**PDF（推荐准备）：**
- 适合：投资人，正式场合
- 优点：专业，格式固定

**Google Docs（可选）：**
- 适合：协作编辑
- 优点：实时更新，评论功能

**转换工具：**
- Markdown → PDF：Pandoc, Typora
- Markdown → Google Docs：复制粘贴 + 格式调整

## 🆘 如果遇到问题

### 常见问题：

**Q: 发送后无人回复？**
A: 
- 等待 7 天再 follow-up
- 尝试不同渠道（邮件 → LinkedIn → Twitter）
- 同时接触更多潜在合作方（不要只等一个）
- 检查邮件是否进垃圾箱

**Q: 他们说"我们再看看"？**
A:
- 要求具体时间表："什么时候方便再次讨论？"
- 提供额外材料："还需要什么信息帮助决策？"
- 保持联系但不催促：每 2 周一次 update

**Q: 他们问的问题我答不上？**
A:
- 诚实："这是个好问题，让我研究一下明天回复你"
- 不要编造答案
- 快速学习和回复
- 展示学习能力也是优势
